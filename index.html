A curated collection of tools, experiments, and ideas where documentation is accessed directly through SSH instead of the browser.

No browser. No dashboards. Just SSH, files, and terminal-native workflows.

What is Docs over SSH?

Docs over SSH is an emerging pattern where documentation is exposed as a navigable filesystem over SSH instead of a traditional web interface.

Instead of browsing pages, you connect via your terminal and explore documentation using familiar commands like ls, cat, and grep.

It turns documentation into something closer to a codebase — terminal-native, scriptable, and AI-agent friendly.

Notable implementations

Supabase

A public SSH-based documentation interface that lets you explore Supabase docs like a filesystem.

Why this matters

  • Terminal-first workflows are becoming more common
  • AI coding agents operate better in CLI environments
  • Reduces context switching between browser and code
  • Documentation behaves like a real filesystem
